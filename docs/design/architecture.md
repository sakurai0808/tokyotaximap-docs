# アプリケーション技術設計

## 採用技術

- フレームワーク: Next.js
- ピンマップ: Leaflet
- コンテンツ: Markdown

## 未決定事項

### CMS

CMSを使用するか、使用する場合に何を採用するかは未決定。

### 全文検索

本文全文、別名キーワード、ミニ単語帳を対象とし、サジェストにも対応する。検索エンジンやインデックスの構築方式は未決定。

### コンテンツデータ

Markdown記事に付与するメタデータの具体的な形式とキー名は未決定。必要となる情報は[コンテンツモデル・編集方針](./content-model.md)に記載する。

### 地図データ

記事とピンの位置情報を紐づける。位置情報の保存場所と管理方法は未決定。
