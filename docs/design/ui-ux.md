# UI・UX設計

## 基本方針

運行中に対象の情報へ素早くアクセスできることを目指す。ピンマップ、検索結果、記事ページ全体に同じUX原則を適用する。

記事ページでは、「車寄せの情報」や「特定方向での右折禁止」など運行中に必要となる可能性が高い情報と、ビジュアルで確認できるYouTubeを優先的に上部へ配置する。

カテゴリごとに優先情報は異なる。例えば「施設」では車寄せ、「交差点」や「首都高」では右折禁止の情報を重視する。これらの「そのページで強調すべき情報を表示したブロック」を「重要ブロック」と呼ぶ。

実践講座、さくいん、ミニ単語帳は教科書や辞書のような位置づけとし、休憩中などの閲覧を想定する。

## 検索UI

### モバイル

トップページにはピンマップを配置し、固定ヘッダーには検索ボタンとハンバーガーメニューを配置する。検索ボタンを独立させ、「すぐに検索できる」ことを示す。固定ヘッダーに置くことで、どのページからでも検索できるようにする。

検索ボタンを押すと、検索ボックスとサジェストに広い領域を使う共通モーダルを開く。文字や候補が狭くなるため、インライン入力欄は採用しない。

参考: ![SP版トップページ](../../logs/ai/images/all/20260619/SP-トップページ.png)

### PC

PCでも検索ボタンをヘッダーに常設する。広い画面ではピンマップの利用も多いと考え、モバイルのような「第一導線: 検索、第二導線: ピンマップ」という優先順位を強く意識しない。

検索モーダルは、すべての画面から共通のものを開く。

## ピンマップ

トップページのファーストビューに常設する。PC、タブレット、スマートフォンではレイアウトのみを変更し、表示コンテンツは変えない。

## 記事ページ

記事ページは次の順序で構成する。

1. 重要ブロック（運行中に読まれる）
2. 「このページをQRコードで共有する」ボタン（運行中に使用される）
3. 本文（主に休憩時など時間のあるときに読まれる）

共有機能は有用だが、QRコードでWebページを共有する行動は一般的ではないため、重要ブロックより表示優先度を下げる。

## ブランドのトーン

教科書的なイメージとする。「緊張感のある実際の運行時の使用」を見据え、オシャレさよりストレスの少ない操作を重視する。

現在のプライマリーカラーは、目に優しく落ち着いた `#95C7EC`。一方で、「安全」「自然」を想起させる緑色も候補として検討中である。

## レスポンシブ対応

モバイルファースト、レスポンシブデザインで設計する。車内での利用を基本とする。

既存サイトのデバイス別使用率（Googleアナリティクス、2026/06/24時点、1か月単位）は次のとおり。

- モバイル: 64.9%
- デスクトップ: 33.8%
- タブレット: 1.4%

対象デバイスはPC、タブレット（TB）、スマートフォン（SP）。タブレットは利用率が低いものの、個人所有の端末を運転席に設置するドライバーもいるため対象とする。それぞれに固有のブレイクポイントを設ける。
